# UI 全面リデザイン PRD

Date: 2026-03-30
Revision: 1

## 現状の問題

1. **Home / Studio / Settings の3ページ構成が分かりにくい** — 非エンジニアはどこで何をするか迷う
2. **英語表記が大量に残っている** — 全て日本語であるべき
3. **サイドバーが重い** — MVP rails、Stage 表記など開発者向け情報が目立つ
4. **Settings が主ナビに並んでいる** — ほぼ見ない情報が同格に並んでいる
5. **Studio が4700行超の巨大ファイル** — 3ペイン構成で情報過多

## ゴール

**アプリを開いたら1画面で全て完結する**シンプルなUI。

## デザイン方針

### 1ページ完結

Home/Studio/Settings の3ページ構成を廃止。**1つのメインページ**に統合する。

### 3ステップの縦フロー

画面を縦に3つのセクションに分け、上から順に進む：

```
┌─────────────────────────────────────┐
│ ヘッダー（アプリ名 + ⚙ アイコン）    │
├─────────────────────────────────────┤
│                                     │
│  ステップ 1: ファイルとやりたいこと    │
│  ┌─────────────────────────────┐    │
│  │ ファイル: [____________] [選択] │    │
│  │ やりたいこと: [テンプレート選択]  │    │
│  │              [自由入力エリア]   │    │
│  │       [準備する →]            │    │
│  └─────────────────────────────┘    │
│                                     │
│  ステップ 2: Copilot に聞く          │
│  ┌─────────────────────────────┐    │
│  │ [依頼をコピー]               │    │
│  │ Copilot の返答:              │    │
│  │ [テキストエリア]              │    │
│  │       [確認する →]            │    │
│  └─────────────────────────────┘    │
│                                     │
│  ステップ 3: 確認して保存            │
│  ┌─────────────────────────────┐    │
│  │ 変更: ○○  行数: ○○  保存先: ○○ │    │
│  │       [保存する]              │    │
│  └─────────────────────────────┘    │
│                                     │
└─────────────────────────────────────┘
```

### 設定は目立たないアイコン

ヘッダー右端に小さな⚙アイコン。クリックでモーダル or ドロワーが開く。

### 全て日本語

ラベル、プレースホルダー、エラーメッセージ、ステータス表示の全てを日本語にする。英語は一切使わない。

### 最近の作業

ヘッダー下に小さく「最近の作業」を表示。クリックで再開できる。普段は邪魔にならない。

### 色とフォント

既存の CSS 変数（暖色系ベージュ + グリーンアクセント）をベースに維持。サイドバーが消えるので全体が明るくなる。

## 画面仕様

### ヘッダー
- 左: アプリアイコン + 「Relay Agent」
- 右: ⚙（設定アイコン）
- 高さ: コンパクト（48px 程度）

### ステップ 1: ファイルとやりたいこと
- ファイル選択（パス入力 + 練習用サンプルチップ）
- やりたいこと（テンプレートチップ + 自由入力テキストエリア）
- ボタン:「準備する」
- ステップ完了後はコンパクトに折りたたまれる（編集可能な状態で）

### ステップ 2: Copilot に聞く
- 「依頼テキストをコピー」ボタン（目立つ）
- 依頼テキストのプレビュー（折りたたみ可能）
- 「Copilot の返答」テキストエリア
- 期待する形式のヒント表示
- ボタン:「確認する」
- ステップ未到達時はグレーアウト

### ステップ 3: 確認して保存
- 3点サマリー（何が変わる / 行数 / 保存先）
- ボタン:「保存する」
- 保存完了後: 結果表示 +「新しい作業を始める」リンク
- ステップ未到達時はグレーアウト

### 設定モーダル
- 実行ポリシー（読み取り専用、プレビュー必須など）
- データの保存場所
- ローカルデータの削除手順

### エラー表示
- 各ステップ内にインライン表示
- 自動修正の通知チップ
- 段階的エラー（JSON構文 / スキーマ / ツール）

## 非ゴール

- バックエンド(Rust)の変更
- IPC インターフェースの変更
- contracts パッケージの変更
- ワークブックエンジンの拡張

## 技術的制約

- SvelteKit + Tauri 2.0 の構成は維持
- ルーティングは `/` のみ使用（studio, settings ルートは削除）
- 既存の IPC 関数（ipc.ts）、continuity.ts、studio-state.ts、auto-fix.ts はそのまま活用
- Svelte 5.0 の writable/derived ストアを使用
