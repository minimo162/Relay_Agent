# Relay Agent — 統合 PRD

最終更新: 2026-03-31

---

## 1. 製品概要

Relay Agent は、Copilot が計画した JSON アクションプランを安全にプレビュー・承認・実行するデスクトップアプリ。
「Copilot は計画、アプリは実行」が基本思想。Copilot API は使わず、ユーザーが Copilot Chat を手動中継する。

- **スタック:** SvelteKit SPA + Tauri v2（Rust バックエンド）
- **対象ファイル:** CSV（完全対応）/ XLSX（inspect + save-copy のみ）
- **対象 OS:** Windows 10/11 x64（NSIS インストーラ）
- **言語:** UI は全て日本語

### 3.2 ステップ動作仕様

**ステップ 1: はじめる**
- ファイル選択（パス入力 + 練習用サンプルチップ）
- やりたいこと（テンプレートチップ + 自由入力テキストエリア）
- ボタン:「準備する」
- **完了後はコンパクトに折りたたまれる（編集可能な状態で）** ← Task 70

**ステップ 2: Copilot に聞く**
- 「依頼テキストをコピー」ボタン（目立つ）
- 依頼テキストのプレビュー（折りたたみ可能）
- 「Copilot の返答」テキストエリア
- 期待する形式のヒント表示
- ボタン:「確認する」
- **ステップ未到達時はグレーアウト** ← Task 69

**ステップ 3: 確認して保存**
- 変更差分の事前確認を常に表示する（必須）
- 「変更前 → 変更後」の差分を行単位で表示（SheetDiff カード）
- 3点サマリー（何が変わる / 行数 / 保存先）
- ボタン:「保存する」（内容確認後に有効化）
- 保存完了後: 結果表示 +「新しい作業を始める」リンク
- **ステップ未到達時はグレーアウト** ← Task 69

### 3.3 設定モーダル

- 実行ポリシー（読み取り専用、プレビュー必須など）

### 3.5 色とフォント

## 4. Copilot JSON やりとり

### 4.1 依頼テキスト（ステップ2でコピーする文章）

以下の構成で Copilot に渡す依頼テキストを生成する:

```
Relay Agent からの依頼です。

1. やりたいこと
- 作業名: {タスク名}
- 目的: {目的テキスト}

2. 対象ファイル
- ファイル: {ファイルパス}
- シート: {シート名}
- 列（使える名前をそのまま使うこと）:        ← Task 71 で追加
  - id (integer)
  - name (string)
  - amount (number)
  ...

3. 使ってよい操作
- table.filter_rows: 条件に合う行だけ残す
- table.rename_columns: 列名を変更する
- ...

4. 回答ルール
- JSON のみを返してください。
- ``` で囲まないでください。
- パス区切りは / を使ってください。
- _ や [ ] を \_ や \[ \] にしないでください。
- tool 名、args 名、列名は見えている文字をそのまま使ってください。
- 上にない tool は使わないでください。

5. 回答テンプレート
{expectedResponseTemplate}

6. 回答例
{テンプレートに合わせた例}                   ← Task 73 で切り替え
```

### 4.2 期待する JSON レスポンス

```json
{
  "version": "1.0",
  "summary": "何をするかを短く説明する",
  "actions": [
    {
      "tool": "table.filter_rows",
      "sheet": "Sheet1",
      "args": {
        "predicate": "[approved] == true"
      }
    },
    {
      "tool": "workbook.save_copy",
      "args": {
        "outputPath": "/actual/path/data.copy.csv"    ← Task 72 で自動生成
      }
    }
  ]
}
```

### 4.3 使えるツール

**read-only:**
- workbook.inspect — ワークブック構造の取得
- sheet.preview — シートのサンプル行表示
- sheet.profile_columns — カラム型とサンプル値の取得
- session.diff_from_base — 差分サマリーの取得

**write:**
- table.rename_columns — 列名変更（args: renames [{from, to}]）
- table.cast_columns — 型変換（args: casts [{column, toType}]）
- table.filter_rows — 行フィルタ（args: predicate）
- table.derive_column — 派生列追加（args: column, expression）
- table.group_aggregate — グループ集計（args: groupBy, measures）
- workbook.save_copy — 別コピー保存（args: outputPath）

**未実装:** join_lookup（将来拡張候補）

### 4.4 auto-fix（自動補正）

Copilot レスポンスの一般的な変形を自動修正するパイプライン。

**実装済み（7種）:**
1. Markdown コードフェンス除去（````json`...````）
2. 前後の空白トリム
3. BOM 除去（U+FEFF）
4. CRLF → LF 正規化
5. Markdown エスケープ除去（`\_`, `\[`, `\]`）
6. 末尾カンマ除去
7. Windows パス変換（`\\` → `/`）

**追加予定（Task 74）:**
8. スマートクォート正規化（`"` `"` → `"` / `'` `'` → `'`）
9. JSON ブロック抽出（prose 前後の除去）
10. 全角スペース除去（U+3000 → 半角スペース）
11. `~~~` フェンス対応

### 4.5 バリデーション段階

| Level | 種類 | 内容 |
|---|---|---|
| 1 | JSON 構文エラー | JSON として parse できない |
| 2 | スキーマエラー | version/summary/actions の不備 |
| 3 | ツールエラー | 許可外の tool 名を使用 |

### 4.6 修正依頼プロンプト（Task 75 で Level 別に強化予定）

バリデーション失敗時に「修正を依頼するテキストをコピー」ボタンで Copilot に送る
修正依頼テキストを生成する。Level ごとに異なる指示を含める。

### 4.7 outputPath 自動生成（Task 72）

入力ファイルパスから推奨 outputPath を自動生成して依頼テキストに埋め込む。

- `/Users/you/data.csv` → `/Users/you/data.copy.csv`
- `/Users/you/report.xlsx` → `/Users/you/report.copy.xlsx`

### 4.8 テンプレート対応の回答例（Task 73）

| テンプレートキー | 回答例に使うツール |
|---|---|
| rename_columns | table.rename_columns + workbook.save_copy |
| cast_columns | table.cast_columns + workbook.save_copy |
| filter_rows | table.filter_rows + workbook.save_copy |
| derive_column | table.derive_column + workbook.save_copy |
| group_aggregate | table.group_aggregate + workbook.save_copy |
| (カスタム) | table.filter_rows + workbook.save_copy |
| (サンプルCSV) | revenue-workflow-demo 専用例 |

## 5. 安全性

- **save-copy only** — 元ファイルは読み取り専用。書き込みは常に別コピーへ。
- **preview 必須** — 実際の write 前に変更予定の差分を必ず表示。
- **approval 必須** — ユーザーが確認してから実行。
- **任意コード実行禁止** — shell、VBA、外部ネットワークアクセスは実装しない。
- **raw Excel formula 拒否** — derive_column は Expr AST 評価 → values_only で書き込み。
- **CSV injection 対策** — 出力時に先頭 `= + - @` の文字列セルを安全化。

## 6. ワークブックエンジン

### CSV（完全対応）
- read / transform / write の全フロー
- 全 write ツール対応

### XLSX（制限付き）
- inspect + save-copy のみ
- write ツールによる変換は未対応
- ワークブック fidelity（マクロ・書式・グラフ）は保証しない

## 7. Inspection Details

### ワークブックアーティファクトブラウザ
選択されたターンの read-only アーティファクトを表示:
- workbook-profile（シート構造・列情報）
- sheet-preview（サンプル行テーブル）
- column-profile（型推論・null 数・サンプル値）
- diff-summary / preview（変更列・影響行数）

### ターンライフサイクル詳細
ターンの各ステップを read-only で閲覧:
- Packet（依頼内容・スコープ要約）
- Validation（pass/fail/warning と理由）
- Approval（承認時刻・対象プレビュー）
- Execution（成功/失敗・出力先・時刻）

temporary mode でも current turn は live state から表示。
reviewer mode では同じ内容を read-only で閲覧可能。
