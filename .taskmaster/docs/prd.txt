# Relay Agent — 統合 PRD

最終更新: 2026-04-01

---

## 1. 製品概要

Relay Agent は、M365 Copilot をエージェントとして活用し、PC 作業全般を自動化するデスクトップアプリ。
ユーザーが目標を伝えると、アプリと Copilot が何度も自動でやりとりしながら処理を進め、
書き込み・削除など不可逆な操作のみ人間が確認する「Human-in-the-Loop エージェントループ」を実現する。

**フェーズ1（現状）:** スプレッドシートの 1 ショット変換（CSV/XLSX）
**フェーズ2（次期）:** 読み取りツール自動実行 + 多ターン Copilot ループ
**フェーズ3（将来）:** ファイル操作・テキスト処理など PC 作業全般への拡張

- **スタック:** SvelteKit SPA + Tauri v2（Rust バックエンド）
- **ブラウザ自動化:** Playwright TypeScript（Edge CDP 接続）
- **対象ファイル:** CSV（完全対応）/ XLSX（inspect + save-copy のみ）
- **対象 OS:** Windows 10/11 x64（NSIS インストーラ）
- **言語:** UI は全て日本語

### 3.2 ステップ動作仕様

- 変更差分の事前確認を常に表示する（必須）

- 3点サマリー（何が変わる / 行数 / 保存先）
- ボタン:「保存する」（内容確認後に有効化）
- 保存完了後: 結果表示 +「新しい作業を始める」リンク
- **ステップ未到達時はグレーアウト** ← Task 69

## 4. Copilot JSON やりとり

### 4.1 依頼テキスト（ステップ2でコピーする文章）

以下の構成で Copilot に渡す依頼テキストを生成する:

```
Relay Agent からの依頼です。

1. やりたいこと
- 作業名: {タスク名}
- 目的: {目的テキスト}

2. 対象ファイル
- ファイル: {ファイルパス}
- シート: {シート名}
- 列（使える名前をそのまま使うこと）:        ← Task 71 で追加
  - id (integer)
  - name (string)
  - amount (number)
  ...

3. 使ってよい操作
- table.filter_rows: 条件に合う行だけ残す
- table.rename_columns: 列名を変更する
- ...

4. 回答ルール
- JSON のみを返してください。
- ``` で囲まないでください。
- パス区切りは / を使ってください。
- _ や [ ] を \_ や \[ \] にしないでください。
- tool 名、args 名、列名は見えている文字をそのまま使ってください。
- 上にない tool は使わないでください。

5. 回答テンプレート
{expectedResponseTemplate}

6. 回答例
{テンプレートに合わせた例}                   ← Task 73 で切り替え
```

### 4.2 期待する JSON レスポンス

```json
{
  "version": "1.0",
  "summary": "何をするかを短く説明する",

### 4.3 使えるツール

**read-only:**
- workbook.inspect — ワークブック構造の取得
- sheet.preview — シートのサンプル行表示
- sheet.profile_columns — カラム型とサンプル値の取得
- session.diff_from_base — 差分サマリーの取得

**write:**
- table.rename_columns — 列名変更（args: renames [{from, to}]）
- table.cast_columns — 型変換（args: casts [{column, toType}]）
- table.filter_rows — 行フィルタ（args: predicate）
- table.derive_column — 派生列追加（args: column, expression）

- workbook.save_copy — 別コピー保存（args: outputPath）

### 4.4 auto-fix（自動補正）

Copilot レスポンスの一般的な変形を自動修正するパイプライン。

**実装済み（7種）:**
1. Markdown コードフェンス除去（````json`...````）
2. 前後の空白トリム
3. BOM 除去（U+FEFF）
4. CRLF → LF 正規化
5. Markdown エスケープ除去（`\_`, `\[`, `\]`）
6. 末尾カンマ除去
7. Windows パス変換（`\\` → `/`）

**追加予定（Task 74）:**
8. スマートクォート正規化（`"` `"` → `"` / `'` `'` → `'`）
9. JSON ブロック抽出（prose 前後の除去）
10. 全角スペース除去（U+3000 → 半角スペース）
11. `~~~` フェンス対応

## 5. 安全性

- **save-copy only** — 元ファイルは読み取り専用。書き込みは常に別コピーへ。
- **preview 必須** — 実際の write 前に変更予定の差分を必ず表示。
- **approval 必須** — ユーザーが確認してから実行。
- **任意コード実行禁止** — shell、VBA、外部ネットワークアクセスは実装しない。
- **raw Excel formula 拒否** — derive_column は Expr AST 評価 → values_only で書き込み。
- **CSV injection 対策** — 出力時に先頭 `= + - @` の文字列セルを安全化。

## 13. Copilot ブラウザ自動化（Tasks 76–83）

### 13.1 背景・目的

現状のステップ2では、ユーザーが手動で以下を行っている：

1. 「依頼をコピー」ボタンでプロンプトをクリップボードにコピー
2. ブラウザで M365 Copilot Chat を開く
3. プロンプトを貼り付けて送信
4. Copilot の返答をコピー
5. アプリの「Copilot の返答」テキストエリアに貼り付け

この手順を Playwright による Edge ブラウザ自動化で代替し、ステップ2を「送信ボタン1回」で完結させる。

### 13.2 技術スタック選定

| 選択肢 | 判断 | 理由 |
|---|---|---|
| **Playwright TypeScript** | ✅ 採用 | 既存スタック（Node.js/pnpm/TS）に完全一致。追加ランタイム不要。`connectOverCDP` でログイン済みEdgeにアタッチ可能。`waitForResponse()` でAPI応答を直接キャプチャできDOMスクレイピングより堅牢。`playwright codegen` でセレクタ自動生成。 |
| browser-use | ❌ 不採用 | LLM APIキーとトークンコストが毎回発生。推論レイテンシ10〜60秒。Pythonランタイム必須でTauri統合が複雑。確定的タスクにAIエージェントは過剰。 |
| Puppeteer | ❌ 不採用 | Playwrightのサブセット。`storageState`・`getByRole`等の安定ロケータがない。コードジェネレータなし。 |
| Python Playwright | ❌ 不採用 | Python 3.11+ランタイムをユーザーに要求する配布負担がある。Node.js版で同等以上のことができる。 |

### 13.3 自動化の対象フロー

```
[ステップ2 UI]
  ↓ 「Copilotに自動送信」ボタン押下
[Tauri @tauri-apps/plugin-shell]
  ↓ Command.create('node', ['scripts/copilot-browser.js', '--prompt', ...])
[scripts/copilot-browser.ts (tsx / node)]
  ↓ playwright.chromium.connectOverCDP('http://localhost:9222')
  │   ※ ユーザーの Edge がログイン済みで CDP が有効化されていることが前提
  ↓ page.goto('https://m365.cloud.microsoft/chat/')
  ↓ 新規チャット開始（getByTestId('newChatButton').click()）
  ↓ プロンプト入力（getByRole('textbox').fill(prompt)）
  ↓ 送信（Enter キーまたは送信ボタン）
  ↓ waitForResponse() で Copilot API の streaming 応答をキャプチャ
     ※ DOMポーリングではなくネットワークレベルで取得 → セレクタ変更の影響なし
  ↓ JSON.stringify({ status: 'ok', response }) を stdout に出力
[ステップ2 UI]
  ↓ stdout をパース → テキストエリアに自動セット → 「確認する」ボタン有効化
```

### 13.4 認証・セッション管理

Edge に CDP デバッグポートを付けてユーザーが手動起動することで、既存の M365 ログイン済みセッションをそのまま利用できる。専用プロファイルの作成・ログイン待機・Cookie 管理が不要になる。

**Edge の CDP 有効化手順（初回のみ・ユーザー操作）:**
```
Microsoft Edge を以下のショートカットで起動（またはタスクスケジューラに登録）:
msedge.exe --remote-debugging-port=9222 --no-first-run
```

設定モーダルでポート番号を変更可能にする。デフォルト: `9222`。

`connectOverCDP` が失敗した場合（Edge が未起動等）はユーザーに起動を促すダイアログを表示する。

### 13.5 レスポンスキャプチャ方式

DOM ポーリングではなく Playwright の `page.waitForResponse()` を使い、Copilot が内部で使う streaming API エンドポイントの応答を直接キャプチャする。

```typescript
const [response] = await Promise.all([
  page.waitForResponse(
    r => r.url().includes('/api/chat') && r.status() === 200,
    { timeout: 60_000 }
  ),
  sendButton.click(),
]);
const body = await response.text();
// streaming JSON (NDJSON / SSE) を parse して最終テキストを抽出
```

エンドポイントが変更されたときのフォールバックとして DOM 取得も保持するが、こちらは補助的な位置づけとする。

### 13.6 UI 変更（ステップ2）

```
ステップ 2: Copilot に聞く
┌──────────────────────────────────────┐
│ [依頼をコピー]  [Copilotに自動送信 ▶]  │  ← 新ボタン追加
│ 依頼テキストのプレビュー（折りたたみ）  │
│ Copilot の返答:                       │
│ [テキストエリア]  ← 自動入力も可能    │
│       [確認する →]                   │
└──────────────────────────────────────┘
```

自動送信中はスピナーを表示し、完了後テキストエリアを自動入力する。エラー時はインラインエラーメッセージを表示し、手動入力にフォールバック可能とする。

### 13.7 エラーハンドリング

| エラー種別 | 対処 |
|---|---|
| CDP 接続失敗（Edge 未起動） | 「Edge を CDP モードで起動してください」ダイアログ + 手動フォールバック |
| ナビゲーション失敗（ログイン画面） | 「M365 にログインしてから再試行してください」メッセージ |
| waitForResponse タイムアウト | DOMポーリングにフォールバック → それも失敗なら手動フォールバック |
| Copilot エラー応答 | エラー内容表示 + 最大2回リトライ |
| セレクタ変更による DOM 不一致 | 設定でセレクタ変更可能 + フォールバックメッセージ |

- Edge 以外のブラウザ対応

## 14. エージェントループ（フェーズ2 — Tasks 84–95）

### 14.1 背景・目的

Claude Cowork（Anthropic、2026/01）や Copilot Cowork（Microsoft、2026/03）が示したように、
「ユーザーが目標を伝えるだけで AI がタスクを完走し、確認が必要な場面のみ人間が介入する」
アーキテクチャが標準になりつつある。

現状の Relay Agent は「Copilot 1回 → 人間が確認 → 実行」という 1 ショット方式。
フェーズ2では、読み取りツールの自動実行と Copilot への結果フィードバックを繰り返す
**多ターン エージェントループ** に進化させる。

### 14.2 ループ設計

```
ユーザー: 目標を入力して「開始」
  │
  ▼
┌─ ターン N ─────────────────────────────────────────┐
│ 1. App → Copilot: タスク説明 + ファイル情報 + 前ターン結果 │
│ 2. Copilot → App: { actions: [...], status: "thinking" } │
│ 3. App: アクションを read / write に分類             │
│                                                    │
│   [read アクション]                                 │
│   → 自動実行（承認不要）                            │
│   → 結果を次ターン用コンテキストに追加               │
│   → ターン N+1 へ（Copilot に結果を返送）            │
│                                                    │
│   [write アクション]                                │
│   → 承認ゲート: SheetDiff カード + 「保存する」ボタン │
│   → ユーザー承認後に実行                            │
│   → ループ終了                                      │
└─────────────────────────────────────────────────────┘
  │
  ▼
完了: 結果表示 + 「新しい作業を始める」
```

### 14.3 Copilot レスポンス スキーマ拡張

既存の `copilotTurnResponseSchema` に `status` フィールドを追加:

```typescript
{
  version: "1.0",
  status: "thinking" | "ready_to_write" | "done" | "error",
  summary: string,           // ユーザー向け進捗説明
  actions: SpreadsheetAction[],
  message?: string,          // Copilot からの補足（エラー詳細等）
  followupQuestions?: string[],
  warnings?: string[]
}
```

| status | 意味 | アプリの動作 |
|---|---|---|
| `thinking` | まだ情報収集中 | read actions を実行してループ継続 |
| `ready_to_write` | 書き込み計画が確定 | write actions を承認ゲートに表示 |
| `done` | 追加アクション不要 | ループ終了・結果表示 |
| `error` | Copilot がエラー | エラーメッセージ表示・手動フォールバック |

### 14.4 ツール実行結果フォーマット

read ツール実行後、Copilot への次ターンプロンプトに以下を追加:

```
【前のターンの実行結果】

▶ workbook.inspect
{シート一覧・行数・列名 JSON}

▶ sheet.preview
{先頭25行のサンプルデータ}

▶ sheet.profile_columns
{列ごとの型推論・null数・サンプル値}

---
上記の結果を踏まえて、次のアクションを JSON で返してください。
書き込み準備が整った場合は status を "ready_to_write" にしてください。
```

### 14.5 ループ安全ガード

| ガード | 設定 | 理由 |
|---|---|---|
| 最大ターン数 | 10 ターン | 無限ループ防止 |
| 1 ターン タイムアウト | 120 秒 | Copilot 無応答対策 |
| read ツール同一引数 重複実行 | 禁止（警告） | 同じ情報を何度も取得するループ検出 |
| write ツール 自動実行 | 禁止（承認必須） | 安全性の絶対保証 |

### 14.6 UI 変更

**ステップ2の進化:**

```
ステップ 2: Copilot と作業中
┌──────────────────────────────────────────┐
│ [ターン 1 / 最大10]  経過時間: 00:23      │
│                                          │
│ ▶ workbook.inspect ... 完了 ✓           │
│ ▶ sheet.preview ... 完了 ✓              │
│ ▶ sheet.profile_columns ... 実行中 ⟳    │
│                                          │
│ Copilot: 「列型を確認しました。          │
│            フィルタ計画を立てています…」 │
│                                          │
│ [キャンセル]                             │
└──────────────────────────────────────────┘
```

write アクションが返ってきたら自動的にステップ3（承認画面）へ遷移。

### 14.7 Copilot への自動送信との連携

セクション13のブラウザ自動化（Playwright + Edge CDP）をループ内で再利用する。
各ターンで `sendToCopilot(prompt)` を呼び出し、返答を受け取る。
ユーザー操作はゼロ（「開始」ボタンのみ）。

## 15. PC 作業ツール拡張（フェーズ3 — Tasks 96 以降）

### 15.1 ファイル操作ツール

スプレッドシート専用を超え、一般的なファイル操作を Rust バックエンドで実装する。

| ツール | 種別 | 内容 |
|---|---|---|
| `file.list` | read | ディレクトリ内のファイル・フォルダ一覧 |
| `file.read_text` | read | テキストファイルの内容取得（最大 1 MB） |
| `file.stat` | read | ファイルのサイズ・更新日時・存在確認 |
| `file.copy` | write | ファイルを別パスにコピー（承認必須） |
| `file.move` | write | ファイルを移動・リネーム（承認必須） |
| `file.delete` | write | ファイル削除（承認必須・ゴミ箱へ） |

### 15.2 テキスト処理ツール

| ツール | 種別 | 内容 |
|---|---|---|
| `text.search` | read | ファイル内のパターン検索（regex 対応） |
| `text.replace` | write | 検索＆置換（承認必須・元ファイルはバックアップ） |

### 15.3 安全ポリシー

- read ツール: 自動実行可（ループ内でそのまま使用）
- write ツール: 常に承認ゲートを経由（エージェントループ §14.2 と同一ルール）
- 実行可能ファイルの起動・シェルコマンド実行: フェーズ3でも実装しない

### 15.4 Contracts 拡張

`packages/contracts/src/file.ts` を新設し、`fileActionSchema` を定義。
`spreadsheetActionSchema` と並列に `relayActionSchema = z.union([spreadsheetActionSchema, fileActionSchema])` を導出する。
