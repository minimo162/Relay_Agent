# Relay Agent — Copilot 実機連携 手動 E2E テストプロンプト

作成日: 2026-04-02
対象: Windows 側 Codex（M365 Copilot ログイン済み環境）

---

## 目的

Relay Agent の全機能を **実際の M365 Copilot と通信しながら** エンドツーエンドで検証する。
WebDriver スモークや Rust 単体テストでは確認できない「Copilot との往復」「CDP 接続」「UI 状態遷移」「ファイル出力」を網羅的にテストし、結果を `docs/E2E_COPILOT_LIVE_REPORT.md` に記録する。

---

## 前提条件

1. **Windows 10/11 x64**
2. **Microsoft Edge** がインストール済み
3. **M365 Copilot** にブラウザでログイン済み（`https://m365.cloud.microsoft/chat/` にアクセスして会話が開始できる状態）
4. **Relay Agent** がビルド済み（`pnpm tauri:build` または `pnpm tauri:dev` で起動可能）
5. テスト用ワークスペースを準備:

```powershell
New-Item -ItemType Directory -Force -Path C:\relay-test
Copy-Item examples\revenue-workflow-demo.csv C:\relay-test\data_a.csv
Copy-Item examples\revenue-workflow-demo.csv C:\relay-test\data_b.csv
Copy-Item examples\revenue-workflow-demo.csv C:\relay-test\data_c.csv

# 空ファイル（エラー系テスト用）
New-Item -ItemType File -Force -Path C:\relay-test\empty.csv
Set-Content C:\relay-test\empty.csv ""
```

6. `cargo test` と `pnpm typecheck` が事前にパスしていること

---

## 実施手順

各シナリオを順番に実施し、チェックボックスを `[x]`（合格）/ `[ ]` + `> 理由`（不合格）/ `[~]` + `> 理由`（スキップ）で埋めること。

## Phase A: CDP 接続とブラウザオートメーション基盤

### A-1 Edge 自動起動 + CDP 接続

1. Relay Agent を起動する（`pnpm tauri:dev`）
2. 設定モーダルを開く（歯車アイコン `設定を開く`）
3. 以下を設定して保存:
   - 「Edge 自動起動」をオン
   - 「CDP ポート」を 9333（デフォルト）
   - 「タイムアウト」を 60000ms

確認項目:
- [ ] 設定保存後にトースト / 確認表示が出ること
- [ ] Edge が CDP ポート付きで自動起動されること（タスクマネージャーで `msedge.exe --remote-debugging-port=9333` を確認）
- [ ] `check_copilot_connection` が成功すること（UI 上に接続状態が緑/正常で表示されること）
- [ ] M365 Copilot のチャット画面（`m365.cloud.microsoft/chat/`）が Edge 内で表示可能なこと

### A-2 CDP 手動接続（自動起動オフ）

1. 設定で「Edge 自動起動」をオフにする
2. Edge を手動で CDP 付きで起動:
   ```powershell
   Start-Process "msedge" "--remote-debugging-port=9333 --user-data-dir=C:\relay-test\edge-profile"
   ```
3. Edge で `https://m365.cloud.microsoft/chat/` を開く

確認項目:
- [ ] Relay Agent の接続確認が成功すること
- [ ] 手動起動した Edge の CDP ポートに接続できること

### A-3 CDP 接続失敗ハンドリング

1. Edge を完全に終了する
2. 設定で「Edge 自動起動」をオフにする
3. ガイドモードでファイルとゴールを入力して送信を試みる

確認項目:
- [ ] エラーメッセージ「Edge に接続できませんでした」が表示されること
- [ ] エラーコード `CDP_UNAVAILABLE` 相当の UI 表示がされること
- [ ] アプリがクラッシュせず、再試行可能な状態であること

### A-4 未ログインハンドリング

1. Edge を CDP 付きで起動するが、M365 からログアウトした状態にする
2. ガイドモードで送信を試みる

確認項目:
- [ ] エラーメッセージ「M365 Copilot にログインしていません」が表示されること
- [ ] エラーコード `NOT_LOGGED_IN` 相当の UI 表示がされること
- [ ] 「Edge で M365 にログインしてから再試行」の案内が表示されること

## Phase B: ガイドモード基本フロー（手動コピペ）

### B-2 ステップ 2 — Copilot 手動コピペ

1. 表示された指示テキストをクリップボードにコピーする

3. Copilot の応答（JSON 形式のアクション定義）をコピーする
4. Relay Agent の応答テキストエリアに貼り付ける
5. 「送信」ボタンを押す

確認項目:
- [ ] Copilot が JSON 形式のアクション応答を返すこと（`{"status": "...", "actions": [...]}`）

- [ ] バリデーション成功後にプレビュー画面に遷移すること
- [ ] ActivityFeed に「Copilot が応答しました」イベントが表示されること

### B-3 ステップ 2（バリデーション失敗時）

1. 応答テキストエリアに不正な JSON（例: `{"invalid": true}`）を貼り付けて送信する

確認項目:
- [ ] バリデーションエラーメッセージが表示されること
- [ ] 「修正プロンプト」が生成され、Copilot に再送信できる案内が表示されること
- [ ] ステップ 2 に留まり、再入力が可能なこと
- [ ] ActivityFeed にエラーイベントが記録されること

### B-4 ステップ 3 — プレビュー・承認・保存

1. B-2 で正常に進んだ状態から、プレビュー画面を確認する

確認項目:
- [ ] 差分プレビュー（追加/削除行数、影響列）が表示されること
- [ ] 「元のファイルはそのまま残ります。変更は別のコピーに保存されます。」メッセージが表示されること
- [ ] 「保存」ボタン（承認ボタン）が表示されること

- [ ] 出力ファイルに approved=true の行のみが残っていること
- [ ] ActivityFeed に「完了」イベントが表示されること
- [ ] CompletionTimeline（完了画面）が表示されること

- [ ] プログレスバーに「Copilot に送信中...」が表示されること
- [ ] `copilot-browser-progress` イベントで接続→送信→応答待ちのステータスが順に表示されること

- [ ] 応答のバリデーションが自動的に行われること

- [ ] プレビュー画面で差分が表示されること

- [ ] 出力ファイルが生成されていること

### C-2 自動送信タイムアウト

1. 設定で「タイムアウト」を 5000ms（5秒）に設定する
2. 長い処理が必要なゴール（例: 「全列のデータ型を分析して統計サマリーを作成して」）を入力して自動送信する

確認項目:
- [ ] タイムアウトエラーメッセージ「Copilot の応答待機がタイムアウトしました」が表示されること
- [ ] 「手動でコピー＆ペーストしてください」のフォールバック案内が表示されること
- [ ] 手動コピペモードにフォールバックできること
- [ ] 設定を元に戻す（60000ms）

### C-3 Copilot エラー応答

1. 意図的に Copilot がエラーを返すゴール（例: 非常に長い無意味な文字列）を送信する

確認項目:
- [ ] `COPILOT_ERROR` または `SEND_FAILED` エラーが適切に表示されること

1. パイプライン・バッチモードをすべてオフにする
2. ガイドモードで `data_a.csv` + 「approved が true の行だけ残して別名保存して」で実行する

確認項目:
- [ ] ゴール入力 → Copilot 自動送信 → プラン承認 → 変換実行 → 保存が完走すること
- [ ] 出力ファイルが正しく生成されていること

### J-2 アプリ再起動後のセッション復元

1. タスクを 1 件完了させる
2. アプリを再起動する
3. セッション一覧を確認する

確認項目:
- [ ] 完了済みセッションが履歴に表示されること
- [ ] セッションの詳細（ゴール、ファイルパス、結果）が復元されていること

### J-3 プロジェクトメモリの永続化

1. タスク完了後、プロジェクトメモリにパターンが学習されていることを確認する
2. アプリを再起動する
3. 同じプロジェクトで新しいタスクを開始する

確認項目:
- [ ] 前回のタスクで学習されたプロジェクトメモリが RelayPacket に反映されていること

6. Phase H（承認ゲート）は Phase C 以降で実施する
7. Phase I（機密データ評価）は独立して実施可能
