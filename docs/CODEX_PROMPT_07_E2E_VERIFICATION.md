# Codex プロンプト 07 — E2E 検証ガイド（Task 95）

## 対象タスク

- **Task 95**: エージェントループ E2E 検証（手動）

## 前提条件

- Tasks 85〜92 の実装が完了していること
- Edge が CDP モードで起動していること（ポート 9222）
- M365 Copilot にログイン済みであること
- `pnpm --filter @relay-agent/desktop tauri:dev` が起動していること
- サンプル CSV が用意されていること（`examples/revenue-workflow-demo.csv`）

---

## 検証シナリオ 1: read-only ループ（ファイル内容確認）

**目的**: 書き込みなしで Copilot が CSV を確認し `done` を返すシナリオ

### 手順

1. アプリを起動し、`examples/revenue-workflow-demo.csv` を選択
2. 目的欄に「このCSVの内容と列の型を教えてください」と入力
3. 「準備する」をクリック（ステップ2へ）
4. 設定モーダルで「エージェントループモード」を ON にする
5. 「Copilotに自動送信 ▶」をクリック

### 期待する動作

- ループログに `workbook.inspect` / `sheet.preview` / `sheet.profile_columns` が順に表示される
- 各ツールが ✓（完了）になる
- Copilot が `status: "done"` を返し、ループが終了する
- エラーなしで完了メッセージが表示される

### 確認ポイント

- [ ] ターン数が 3 以内で完了する
- [ ] ループログが正しく表示される
- [ ] 「保存する」ボタンが表示されない（write actions なし）
- [ ] 元ファイルが変更されていない

---

## 検証シナリオ 2: 多ターン → write で完了

**目的**: Copilot が情報収集後に書き込み計画を立てるシナリオ

### 手順

1. アプリを起動し、`examples/revenue-workflow-demo.csv` を選択
2. 目的欄に「approved が true の行だけ残し、amount の合計を追加列として保存してください」と入力
3. 「準備する」をクリック
4. 「エージェントループモード」を ON にする
5. 「Copilotに自動送信 ▶」をクリック

### 期待する動作

- ターン1: `workbook.inspect` / `sheet.preview` / `sheet.profile_columns` が自動実行される
- Copilot がデータを確認し `status: "thinking"` を返す
- ターン2: 追加情報があれば取得、または `status: "ready_to_write"` を返す
- `table.filter_rows` + `table.derive_column` + `workbook.save_copy` が write actions として届く
- 自動的にステップ3（承認画面）に遷移し、SheetDiff カードが表示される
- 「保存する」で保存が完了する

### 確認ポイント

- [ ] 2〜3ターンで `ready_to_write` に到達する
- [ ] ステップ3に自動遷移する
- [ ] SheetDiff カードに変更内容が表示される
- [ ] 保存後に出力ファイルが生成される
- [ ] 元ファイルが変更されていない

---

## 検証シナリオ 3: キャンセル

### 手順

1. エージェントループを開始する
2. ループ実行中に「キャンセル」ボタンをクリック

### 期待する動作

- ループが停止する
- エラーメッセージが表示されない（または「キャンセルされました」と表示）
- ステップ2の状態に戻る
- 手動送信モードに切り替えられる

### 確認ポイント

- [ ] キャンセル後にUIが操作可能になる
- [ ] 出力ファイルが生成されていない

---

## 検証シナリオ 4: 最大ターン数到達

### 手順

1. 設定モーダルで最大ターン数を 2 に設定
2. 複雑な目的（複数回の情報収集が必要）でループを開始
3. 2ターン後に最大到達エラーが出ることを確認

### 期待する動作

- 「最大ターン数（2）に達しました」エラーが表示される
- 手動フォールバック（「手動入力に切り替え」リンク）が表示される

---

## 検証シナリオ 5: エラー状態（CDP 未接続）

### 手順

1. Edge を CDP なしで起動しておく（または Edge を閉じる）
2. エージェントループを開始する

### 期待する動作

- 「Edge が CDP モードで起動していません」エラーが表示される
- エージェントループが停止する
- 手動モードに切り替え可能

---

## 問題が発生した場合

以下のログを確認する:

1. **DevTools コンソール** (F12): `[copilot-browser]` および `[agent-loop]` プレフィックスのログ
2. **Rust ログ**: `RUST_LOG=debug pnpm --filter @relay-agent/desktop tauri:dev`
3. **Copilot レスポンス**: DevTools の close code / stdout ログ
